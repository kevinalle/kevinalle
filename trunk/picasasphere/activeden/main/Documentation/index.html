Using PhotoSphere - top

To include PhotoSphere in your webpage you just have to embed the swf in the body of your document.

Upload photosphere.swf to your server and make sure its in the same directory as the html file that will embed it.
Edit you HTML and add something like the following wherever you want to include PhotoSphere:

		

<object classid="clsid:d27cdb6e-ae6d-11cf-96b8-444553540000" width="400" height="400" id="picasa" align="middle"> 
	<param name="movie" value="picasasphere.swf?service=picasa&user=<YOUR_PICASA_USERNAME>&album=<YOUR_ALBUM>&max=30&thumbquality=5"/>
	<param name="allowScriptAccess" value="always" /> 
	<param name="quality" value="high" /> 
	<param name="scale" value="noscale" /> 
	<param name="salign" value="lt" /> 
	<param name="bgcolor" value="#ffffff"/> 
	<param name="wmode" value="transparent"/> 
	<embed src="picasasphere.swf?service=picasa&user=<YOUR_PICASA_USERNAME>&album=<YOUR_ALBUM>&max=30&thumbquality=5" bgcolor="#ffffff" width="400" height="400" name="picasa" quality="high" align="middle" allowScriptAccess="always" type="application/x-shockwave-flash" pluginspage="http://www.macromedia.com/go/getflashplayer" wmode="transparent"/> 
</object>

Save your file.
That's it!

Continue to the next sections to learn how to set the photo sources.

Picasa service - top

The easiest way to use PhotoSphere is to upload your photos to Picasa and have them dynamically served

You can use PhotoSphere to display any public picasaweb user or album.
if you specify an album, the photo thumbnails of that album will be displayed. If not, thumbnails of that users public albums will be displayed.
Every thumbnail will link to the corresponding page in picasaweb

To use PhotoSphere with a picasa album or user, include the corresponding parameters when embedding the swf.
Note the parts of the code where you see this:

picasasphere.swf?service=picasa&user=<YOUR_PICASA_USERNAME>&album=<YOUR_ALBUM>&max=30&thumbquality=5

Replace <YOUR_PICASA_USERNAME> and <YOUR_ALBUM> with your data.
You can exclude the album parameter.

Important Notes: You must specify the parameters TWICE. In the param "movie" and in the embed src.
Please make sure the selected album is public.

Other parameters:
You can also specify the following parameters:
max: The maximum number of thumbnails to display. (The first n photos will be shown)
thumbquality: The quality of the thumbnails. Use a valua between 1 and 8 (5 is default). The qualities correspond to the sizes: 32px, 48px, 64px, 72px, 104px, 144px, 150px and 160px respectively.

Custom xml - top

If you don't want to upload your photos to picasa you can specify the photos to be displayed in an xml file.

The structure of the xml file should be the following:

<photos>
    <photo src="photo1.jpg" url="http://..." />
    <photo src="photo2.jpg" url="http://..." />
    .
    .
    .
</photos>

where src is the url of the thumbnail and href is the location that opens on click.

To use PhotoSphere with your custom url embed it as photosphere.swf?service=custom&xml=photos.xml, where photos.xml is the name of your xml file, and is in the same directory

NOTE: Due to Security reasons, the swf wont work on your local filesystem. you have to upload it to a server in order to make ir work

Source Code - top

You shouldn't need to recompile the swf, but if you want here is how:

The sources are written in HaXe (http://haxe.org).
Make sure you have HaXe installed correctly in your computer.
You can compile it by simply running
haxe compile.hxml
(or by double-clicking compile.hxml in windows)

Also, the sources are in ActionScript3 format in the folder AS3.
You should be able to compile it with the Flex SDK by running
mxmlc -output photosphere.swf __main__.as
(note: the sources were generated with HaXe's AS3 code generator, http://haxe.org/doc/flash/as3gen)

A Flex source file is also included in AS3/photosphere.mxml.
You can compile it with Flex SDK or oen it with FlexBuilder.
